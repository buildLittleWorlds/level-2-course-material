# Parallel Example: Uncertainty-First Symptom Explainer

This packet models how a medical-AI Space idea could become a cautious research journal and paper-lite draft. It is student-facing, but it is not medical advice and it does not describe the student's actual work.

## Core idea

The imagined Space is an **Uncertainty-First Symptom Explainer**. A user enters a short symptom description. The Space responds in two possible interface styles:

- A single-answer style that gives one likely explanation
- A ranked-and-cautious style that gives multiple possibilities, uncertainty language, and a "what to do next" section

The research focus is the interface, not diagnosing patients.

## Research question

When a medical AI tool is built for ordinary users, does a ranked-and-cautious interface communicate uncertainty better than a single-answer style interface?

## Space idea

Inputs:

- Symptom description
- Optional context such as duration and age range
- Interface mode: single-answer or ranked-and-cautious

Outputs:

- Possible explanations in plain language
- Clear uncertainty statements
- Red-flag guidance telling users when to seek professional care
- A short self-check asking whether the output sounded too certain

## What this packet demonstrates

- How to keep a medical-AI project focused on communication and safety rather than diagnosis
- How to use sources to support caution instead of hype
- How to write limitations honestly in a high-stakes domain
- How to make interface design itself into a research question

## How to use this example

Read `sources.md` before the paper. It shows how to find medical-AI sources without turning them into fake evidence. Then read `research-journal.md` and `PAPER.md` to see how the example keeps the claims narrow.

The Code: Generate Function

def generate_text(prompt, temperature, top_p, max_length):
    if not prompt or not prompt.strip():
        return "Type a prompt above first!"

    result = generator(
        prompt,
        temperature=max(temperature, 0.01),
        top_p=top_p,
        max_length=int(max_length),
        do_sample=True,
        num_return_sequences=1,
    )
    return result[0]["generated_text"]

Three sliders feed into one function. Each one changes how the AI writes.

The Code: Sliders

demo = gr.Interface(
    fn=generate_text,
    inputs=[
        gr.Textbox(lines=3, placeholder="Start your text here...",
                   label="Prompt"),
        gr.Slider(minimum=0.1, maximum=2.0, value=0.7,
                  step=0.1, label="Temperature (creativity)"),
        gr.Slider(minimum=0.1, maximum=1.0, value=0.9,
                  step=0.05, label="Top-p (diversity)"),
        gr.Slider(minimum=20, maximum=200, value=100,
                  step=10, label="Max Length (words-ish)"),
    ],
    outputs=gr.Textbox(label="Generated Text", lines=10),
    title="Text Playground",
)

gr.Slider — each one is a knob we hand to the user.

Experiment Rules

Change ONE slider at a time.

"If you change two sliders at once, can you tell which one caused the change?"

No — this is why scientists change one variable at a time.

Experiment 1: Temperature

Prompt: "The secret ingredient in the recipe was"
0.1 — repetitive, safe
0.5 — sensible, focused
0.7 — balanced
1.0 — creative
1.5 — chaotic
2.0 — gibberish?

Experiment 2: Top-p

Same prompt, temperature fixed at 0.7
Top-p = 0.1 — only the top 1–3 most likely words considered
Top-p = 0.5 — moderate vocabulary
Top-p = 0.9 — most words in play
Top-p = 1.0 — everything is possible

Experiment 3: Max Length

Same prompt, same settings — change only length
20 tokens — barely a sentence
50 tokens — a short paragraph
100 tokens — getting longer
200 tokens — does it stay coherent?

Longer isn't always better. Models can lose the thread.

Name the Concept

same MODEL + same PROMPT + different HYPERPARAMETERS
→ different OUTPUT

"Hyper" = above/beyond. These are parameters that sit above the model's learned weights.
The model has millions of internal parameters we can't change.
Hyperparameters are the knobs we control.

Best Settings for the Job

There's no single "best" setting. It depends on the task.

Scary story opening — high creativity, medium length
Formal email to a teacher — low creativity, controlled length
Funny random story — high creativity, longer length
News headline continuation — low creativity, short length

You're learning the universal remote, not just one TV.

Vocabulary

Temperature — controls randomness; low = predictable, high = creative/chaotic
Top-p (nucleus sampling) — limits which words the model considers; low = only top picks
Max tokens — how much text the model generates (tokens ≈ words, roughly)
Sampling — the process of randomly picking the next word from a probability distribution
Hyperparameter — a setting we control at runtime (not learned during training)

Run it a few more times.

Why?

The Missing Control

Riley’s Space has Temperature and Max Tokens — but no repetition penalty.

Without a penalty, the model has no reason NOT to repeat itself.
“I just wrote cardinal. What’s most likely next? Cardinal again.”

This isn’t a bug. It’s a missing control.

The Fix: Two Parameters

repetition_penalty = 1.2
Every word you already said is 20% less likely to be picked again
1.0 = no penalty · 2.0 = aggressive
no_repeat_ngram_size = 2
No two-word phrase can appear twice
A hard rule, not a probability
Riley has two versions of Space 1:
Without penalty → loops and degenerates
With penalty → text actually progresses

Hidden Defaults

The controls you don’t set matter as much as the ones you do.
ChatGPT doesn’t repeat itself — because OpenAI already set a penalty for you
Claude writes without looping — there’s a setting behind the scenes
Your Space looped — because nobody told it not to

You found a missing setting, diagnosed the problem, and fixed it.
That’s what AI engineers do.

Code Walkthrough: Add the Function

def generate_text(prompt, temperature, top_p, max_length):
    if not prompt or not prompt.strip():
        return "Type a prompt above first!"

    result = generator(
        prompt,
        temperature=max(temperature, 0.01),
        top_p=top_p,
        max_length=int(max_length),
        do_sample=True,
        num_return_sequences=1,
    )
    return result[0]["generated_text"]

Three sliders feed into one function.
Each one changes how the AI writes.

A Different Kind of Generation

Summarizer

This model doesn’t create from scratch — it condenses.

Two Kinds of Generation

Text Generator
Short prompt → more text
distilgpt2
Controls: temperature, top-p, max length, repetition penalty
Summarizer
Long article → short summary
distilbart-cnn-12-6
Controls: max length, min length

Both are generative — they produce new text.
Different controls because different jobs.

Code: Summarizer Function + Interface

def summarize(text, max_length, min_length):
    if not text or not text.strip():
        return "Paste some text above first!"

    result = summarizer(
        text,
        max_length=int(max_length),
        min_length=int(min_length),
        do_sample=False,
    )
    return result[0]["summary_text"]

demo = gr.Interface(
    fn=summarize,
    inputs=[
        gr.Textbox(lines=10, placeholder="Paste a paragraph or article...",
                   label="Text to Summarize"),
        gr.Slider(minimum=30, maximum=200, value=130,
                  step=10, label="Max Length"),
        gr.Slider(minimum=10, maximum=100, value=30,
                  step=10, label="Min Length"),
    ],
    outputs=gr.Textbox(label="Summary", lines=5),
    title="Text Summarizer",
)

demo.launch()

Name the Concept

HYPERPARAMETERS

Hyper = above. These sit above the model’s learned weights.
You control them at runtime.

🎸

The Guitar
Wood, strings, shape
= the model’s weights (fixed)

🎛

The Amp Settings
Volume, gain, tone
= hyperparameters (you control these)

The Real-World Connection

Every AI tool you’ve ever used has these same controls.
ChatGPT — has a temperature setting behind the scenes
Claude API — developers set temperature and top_p on every call
Image generators — guidance scale is the same idea

Most products hide the sliders. You just saw what’s behind the curtain.

Research Lens: Parameter Sweep

Changing one variable at a time — that’s experimental design.

Same method scientists use. Hold everything constant, change one thing, observe.

Riley did this: same prompt, different temperatures.
Conclusion: “1.0 produces the best nature writing.”
That’s a parameter sweep.

Vocabulary

Temperature — controls randomness; low = predictable, high = creative/chaotic
Top-p (nucleus sampling) — limits which words the model considers
Repetition Penalty — taxes reused words; 1.2 = 20% less likely per reuse
N-gram — a sequence of N words; no_repeat_ngram_size=2 bans repeated pairs
Hyperparameter — a setting we control at runtime (not learned during training)
Parameter sweep — changing one variable at a time to isolate its effect
Summarization — generating a shorter version of a longer text

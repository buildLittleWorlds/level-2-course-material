Code Walkthrough: Add the Function

def generate_text(prompt, temperature, top_p, max_length):
    if not prompt or not prompt.strip():
        return "Type a prompt above first!"

    result = generator(
        prompt,
        temperature=max(temperature, 0.01),
        top_p=top_p,
        max_length=int(max_length),
        do_sample=True,
        num_return_sequences=1,
    )
    return result[0]["generated_text"]

Three sliders feed into one function.
Each one changes how the AI writes.

Code Walkthrough: Add the Sliders

demo = gr.Interface(
    fn=generate_text,
    inputs=[
        gr.Textbox(lines=3, placeholder="Start your text here...",
                   label="Prompt"),
        gr.Slider(minimum=0.1, maximum=2.0, value=0.7,
                  step=0.1, label="Temperature (creativity)"),
        gr.Slider(minimum=0.1, maximum=1.0, value=0.9,
                  step=0.05, label="Top-p (diversity)"),
        gr.Slider(minimum=20, maximum=200, value=100,
                  step=10, label="Max Length (words-ish)"),
    ],
    outputs=gr.Textbox(label="Generated Text", lines=10),
    title="Text Playground",
)

demo.launch()

gr.Slider — each one is a knob we hand to the user.

Demo Build 2

Summarizer

A different kind of generation.
This model doesn’t create from scratch — it condenses.

Why This Is Generation

Classification
Sorts into buckets
Text Generator
Creates new text
Summarizer
Reads everything, writes shorter

All three are different — but the summarizer is generative, not classification.
It produces new text. It doesn’t just pick a label.

Code Walkthrough: Summarizer

import gradio as gr

distilbart-cnn-12-6 — trained on news articles.
Fast enough for free CPU.

Code: Summarizer Function + Interface

def summarize(text, max_length, min_length):
    if not text or not text.strip():
        return "Paste some text above first!"

    result = summarizer(
        text,
        max_length=int(max_length),
        min_length=int(min_length),
        do_sample=False,
    )
    return result[0]["summary_text"]

demo = gr.Interface(
    fn=summarize,
    inputs=[
        gr.Textbox(lines=10, placeholder="Paste a paragraph or article...",
                   label="Text to Summarize"),
        gr.Slider(minimum=30, maximum=200, value=130,
                  step=10, label="Max Length"),
        gr.Slider(minimum=10, maximum=100, value=30,
                  step=10, label="Min Length"),
    ],
    outputs=gr.Textbox(label="Summary", lines=5),
    title="Text Summarizer",
)

demo.launch()

Name the Concept

HYPERPARAMETERS

Hyper = above. These sit above the model’s learned weights.
You control them at runtime.

🎸

The Guitar
Wood, strings, shape
= the model’s weights (fixed)

🎛

The Amp Settings
Volume, gain, tone
= hyperparameters (you control these)

The Real-World Connection

Every AI tool you’ve ever used has these same controls.
ChatGPT — has a temperature setting behind the scenes
Claude API — developers set temperature and top_p on every call
Image generators — guidance scale is the same idea

Most products hide the sliders. You just saw what’s behind the curtain.

Research Lens: Parameter Sweep

Changing one variable at a time — that’s experimental design.

Same method scientists use. Hold everything constant, change one thing, observe the result.

You did this tonight: same prompt, same model, different temperature → different output.
That’s a parameter sweep.

Vocabulary

Temperature — controls randomness; low = predictable, high = creative/chaotic
Top-p (nucleus sampling) — limits which words the model considers; low = only top picks
Max Length — how much text the model generates (tokens ≈ words, roughly)
Hyperparameter — a setting we control at runtime (not learned during training)
Parameter — a value the model learned during training (we can’t change these)
Summarization — generating a shorter version of a longer text

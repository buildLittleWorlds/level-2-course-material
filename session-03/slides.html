The Code: clean_text()

A function that cleans messy input before the model ever sees it.

def clean_text(text):
    # 1. Strip leading/trailing whitespace
    text = text.strip()

    # 2. Collapse multiple spaces into one
    text = re.sub(r' {2,}', ' ', text)

    # 3. Limit repeated characters ("sooooo" → "soo")
    text = re.sub(r'(.)\1{2,}', r'\1\1', text)

Each step fixes one specific kind of noise.

The Code: Emoji + ALL CAPS

    # 4. Remove emoji
    text = re.sub(
        r'[\U0001F600-\U0001F64F'
        r'\U0001F300-\U0001F5FF'
        r'\U0001F680-\U0001F6FF'
        r'\U00002600-\U000026FF]+',
        ' ', text
    )

    # 5. Normalize ALL CAPS to Title Case
    words = text.split()
    caps_count = sum(1 for w in words
                     if w.isupper() and len(w) > 1)
    if caps_count > 3:
        text = text.title()

When you text 💀💀💀, that IS the meaning. Removing it doesn't help the model — it removes information the model never had.

The Big Question

What You Know That the Model Doesn't

Who sent it — your best friend vs. your teacher
What happened before — the conversation, the context
How they usually talk — their voice, their patterns
Punctuation clues — "fine." is very different from "fine!"

The model has none of this. It just has the words.

CLEAR Framework

A structure for writing good prompts to AI coding assistants.

C — Context — "I have a Gradio app that uses a sentiment model..."
L — Language — "The code is in Python, using the transformers library..."
E — Explain — "When I paste sarcastic text, the model misreads the tone..."
A — Ask — "Can you add a text cleaning function that..."
R — Requirements — "It should handle emoji, repeated characters, and ALL CAPS."

Research Lens

We tested adversarial inputs and performed data preprocessing — cleaning noise vs. signal.

Adversarial testing = deliberately trying to break a system to find its limits.
Preprocessing = transforming data before the model sees it — exactly what clean_text() does.

Research question: What types of text input cause sentiment models to fail, and can preprocessing fix them?

Vocabulary

Data cleaning — preprocessing text to remove noise before the model sees it
Noise — stuff in the input that confuses the model (emoji, extra spaces, weird formatting)
Adversarial input — text deliberately designed to confuse or break a model
Tone — the way something is said — meaning that lives between the words
Feature engineering — transforming raw input into something a model can work with better

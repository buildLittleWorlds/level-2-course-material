Build: World Headlines Space

Connecting to the Real World

AI tools need external data. Real-time headlines are one example.

The Pattern:

  • Import requests
  • Build API parameters
  • Make the call
  • Parse JSON

The fetch_headlines Function

def fetch_headlines(api_key, country, max_articles=10):
	url = "https://gnews.io/api/v4/top-headlines"
	params = {
		"country": country,
		"token": api_key,
		"max": max_articles
	}
	response = requests.get(url, params=params)
	if response.status_code == 200:
		return response.json()
	else:
		return None

# Call it for multiple countries
countries = ["us", "gb", "de", "fr", "jp", "br", "in", "mx", "ke", "au"]
all_headlines = {}
for country in countries:
	all_headlines[country] = fetch_headlines(API_KEY, country)

Teach the model your categories.

Test 1: Tone Deafness

Good news vs. Bad news on breaking headlines

  • Example headlines:
  • "Markets soar as thousands lose jobs"
  • "Record unemployment claims, stock rally"
  • "Disaster brings economic boost"

Notice: measured language about disasters. But which emotion does the model pick?

Failure Mode #1: Tone Deafness

The model misses meaning that IS there.

It reads the words

but misses the story.

What words is it weighting? Why doesn't it see the conflict?

Test 2: Emotional Flattening

Hopeful vs. Worrying vs. Mixed

  • Complex stories:
  • "New climate agreement signed, but targets still too weak"
  • "Peace deal reached despite ongoing violence"
  • "Economy grows while inequality widens"

Real news is rarely one thing.

Failure Mode #2: Emotional Flattening

The model oversimplifies meaning that's complex.

It's forced to pick a winner

when the real answer is "it's complicated."

Can a classification model handle "both/and"? Or is it always "either/or"?

Test 3: Anthropomorphic Projection

Neutral vs. Editorial vs. Clickbait

  • Market & weather metaphors:
  • "Markets panic over rate hike"
  • "Storm rages across region"
  • "Dollar crashes in afternoon trading"

Who is feeling these emotions?

Failure Mode #3: Anthropomorphic Projection

The model invents meaning that ISN'T there.

Markets don't panic.

Storms don't rage with anger.

Is the model reading metaphor? Or just pattern-matching emotions?

Three Failure Modes

Tone Deafness

Misses meaning that IS there

Emotional Flattening

Oversimplifies meaning that's complex

Anthropomorphic Projection

Invents meaning that ISN'T there

Even custom categories didn't fix any of these.

Better labels isn't better understanding.

CLEAR Framework

A Prompt Engineering Pattern

C — Context: What's the situation?

L — Language: What tools/languages?

E — Expectation: What's the goal?

A — Action: What should the model do?

R — Result: How should it look?

CLEAR With News Headlines

C — "I have a Gradio app that fetches news headlines from an API every hour."

L — "Python, using gradio, requests, and transformers."

E — "I want to add sentiment analysis to each headline."

A — "Can you add a function that runs each headline through a sentiment model?"

R — "Use distilbert sentiment. Show colored badges — green for positive, red for negative."

The Old Way: clean_text()

For decades, AI researchers spent most of their time doing exactly this — manually cleaning input so narrow models could function. 
def clean_text(text):
    # 1. Strip leading/trailing whitespace
    text = text.strip()

    # 2. Collapse multiple spaces into one
    text = re.sub(r' {2,}', ' ', text)

    # 3. Limit repeated characters ("sooooo" → "soo")
    text = re.sub(r'(.)\1{2,}', r'\1\1', text)

Each step fixes one specific kind of noise.

The Code: Emoji + ALL CAPS

Test 1: The Sarcastic Narrator

A student describes the worst Monday of their life.
Every sentence sounds positive.

Paste the story into all four Spaces.
According to these models, did this person have a good day?

Failure Mode #1

Tone Deafness

The model misses meaning that IS there.

It hears the notes but misses the music.

Test 2: The Mixed-Emotion Story

Getting into your dream college means leaving home.
Every paragraph has two real emotions at the same time.

Is there a single paragraph where the narrator feels only one thing?

Failure Mode #2

Emotional Flattening

The model oversimplifies meaning that's complex.

It's like being asked "are you happy or sad?" when the answer is both.

Test 3: The Earth Doesn't Feel Anything

A volcanic eruption. Destruction. Regrowth.
Zero human characters.

The models draw emotional arcs. But who is feeling these emotions?
Point to the character who is afraid.

Failure Mode #3

Anthropomorphic Projection

The model invents meaning that ISN'T there.

It finds feelings where there are none — because the words sound emotional.

Three Failure Modes

Tone Deafness
Misses meaning that IS there
(sarcasm, irony)
Emotional Flattening
Oversimplifies complex meaning
(mixed feelings → one label)
Anthropomorphic Projection
Invents meaning that ISN'T there
(emotions in rocks)
28 emotion categories didn't fix any of these.
More labels isn't more understanding.

It Goes Deeper Than English

Remember Ekman's 6 basic emotions from last week? Some researchers say he got it wrong.

Some feelings don't have English words:

Schadenfreude (German) — pleasure at someone else's misfortune
Amae (Japanese) — the sweetness of depending on someone who loves you
Saudade (Portuguese) — deep longing for something absent
Gigil (Filipino) — the urge to squeeze something unbearably cute

The Deeper Problem

If emotion itself is shaped by culture and language, then the problem isn't just "not enough labels."

No set of English labels can capture what every human feels.

This is the wall. More categories can't fix it. The whole approach has a ceiling.

The Wall

For decades, this was AI:
Build a small model. Clean the data by hand. Test it. Watch it fail on anything new. Clean harder. Hit the same wall.

The model reads words.
You read meaning.
No amount of cleaning closes that gap.

So here's my question for next week:

What if we stopped sorting things into buckets?

What if we asked a model to create something new?

To write, instead of read?

CLEAR Framework

A structure for writing good prompts to AI coding assistants.

C — Context — "I have a Gradio app that uses a sentiment model..."
L — Language — "The code is in Python, using the transformers library..."
E — Explain — "When I paste sarcastic text, the model misreads the tone..."
A — Ask — "Can you add a text cleaning function that..."
R — Requirements — "It should handle emoji, repeated characters, and ALL CAPS."

CLEAR in Action

Let's try it live with Claude or ChatGPT:

Context: I have a Hugging Face Space that uses a sentiment model to read the mood of text.

Language: Python, using gradio, transformers, and re.

Explain: When users paste sarcastic or messy text (emoji, ALL CAPS, mixed signals), the model misreads the tone.

Ask: Add a clean_text() function that preprocesses the input.

Requirements: Strip whitespace, collapse repeated characters, remove emoji, normalize ALL CAPS. Show before and after.

Vocabulary

Classification — a model that sorts inputs into predefined categories
Adversarial input — input deliberately designed to confuse or break a model
Tone deafness — the model misses meaning that IS there
Emotional flattening — the model oversimplifies complex meaning into one label
Anthropomorphic projection — the model invents meaning that ISN'T there
Comparative analysis — testing the same question with different tools and comparing results
Operationalization — the specific way you define and measure a concept (e.g., "emotion" as 2 labels vs. 7 vs. 28)
CLEAR Framework — a structure for prompting AI coding assistants

Inside the Space

Every Space has a Files tab. Let's look at what's inside the Mood Meter.

Two files make this whole app work:

app.py — the code that runs the Space
requirements.txt — the packages it needs

That's it. Two files. You don't need to write any code tonight — but understanding what's inside will make you a sharper critic.

What's in app.py?

import gradio as gr
from transformers import pipeline

analyzer = pipeline("sentiment-analysis",
    model="distilbert-base-uncased-finetuned-sst-2-english")
gradio
Builds the web interface — the page you see
pipeline
Loads the AI model — the brain that reads feelings

When you look at any Space's Files tab, you'll see this same pattern.

The Pattern

INPUT → MODEL → OUTPUT

Every AI tool you've ever used follows this pattern.

ChatGPT: your prompt goes in, text comes out.
Image generators: a description goes in, a picture comes out.
The Mood Meter: text goes in, a feeling comes out.

The Mood Meter, Broken Down

TEXT → SENTIMENT MODEL → MOOD READING

Quick check: can you name the input, model, and output for the Emoji Mood Translator?

What's "Sentiment Analysis"?

The model distilbert-base-uncased-finetuned-sst-2-english was trained on movie reviews.

Someone labeled thousands of movie reviews as POSITIVE or NEGATIVE. The model learned the patterns of each.

Now it reads any text through that lens —
even diary entries, song lyrics, and text messages.

The Big Question

Can a machine actually feel, or is it just matching patterns?

But you have a giant rulebook.

Someone slides a Chinese sentence under the door.

The person outside thinks you understand Chinese.
But do you?
